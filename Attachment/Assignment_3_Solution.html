设计要求

  1. 当没有乘客通过时，系统处于待机状态。
  2. 当乘客成功完成支付后，支付终端会在 PAYTERM 信号上输出高电平，以通知系统允许通行；否则，支付终端输出低电平。
    PAYTERM
    低电平：待机
    高电平：支付成功
  3. 门禁系统在接收到支付终端的高电平信号后，应立即打开闸门。若乘客通过完成，则闸门关闭。闸门控制信号 GATE_OPEN 为高电平时，闸门打开；为低电平时，闸门关闭。
    闸门
    低电平：闸门关闭
    高电平：闸门打开
  4. 系统中有 3 个红外传感器 IR1 到 IR3，用于检测乘客是否成功通过。若某个红外传感器被乘客遮挡，无法接收到红外光源，则其输出为高电平；否则输出为低电平。
    IRx
    低电平：未遮挡
    高电平：被遮挡
  5. “成功通过”定义为：乘客在 1 分钟内进入并离开门禁系统。乘客进入，表示传感器 IR1 到 IR3 按顺序依次被遮挡。乘客离开，表示传感器 IR1 到 IR3 按顺序依次重新暴露于红外光源，即依次恢复为未遮挡状态。
  6. 儿童乘客在任意时刻可能只会遮挡 1 个传感器。成年乘客在任意时刻最多会同时遮挡 2 个相邻传感器；携带行李的乘客在任意时刻最多可能同时遮挡全部 3 个传感器。
  7. 乘客有 1 分钟时间完成通行。若超时且乘客仍在闸机内，系统应报警。系统输出 ALARM 高电平表示报警，低电平表示清除报警。或者，若超时但没有乘客正在通过，则闸门关闭。报警会在乘客离开传感器检测区域后被清除。
    报警
    低电平：报警清除
    高电平：报警
  8. 当定时器的使能端 TIMER_ON 置为高电平时，定时器开始计时；当达到 1 分钟时，在 TIME_OUT 引脚输出高电平。当 TIMER_ON 为低电平时，定时器复位，同时 TIME_OUT 输出低电平。
    TIMER_ON
    低电平：复位定时器并清除 TIME_OUT
    高电平：启动定时器
    TIME_OUT
    低电平：未超时
    高电平：已超时

1.设计思路

本解答采用分层有限状态机，与作业提示相匹配。 完整系统分为三个部分：

  • 主有限状态机：处理待机、闸门控制、超时和报警。
  • 通行检测子有限状态机：检查乘客是否按正确顺序通过三个红外传感器。
  • 1 分钟定时器：作为由 TIMER_ON 控制的外部模块。

这种做法比把所有行为强行放入一个大型统一有限状态机更清晰。

2.(a) 有限状态机的输入和输出

2.1 外部系统信号

输入

  • PAYTERM：1 表示支付成功，0 表示待机。
  • IR1、IR2、IR3：红外传感器输出。
  • TIME_OUT：1 分钟定时器的输出。
  • CLK：同步有限状态机实现所需的时钟输入。
  • RESET：复位输入。

输出

  • GATE_OPEN：1 打开闸门，0 关闭闸门。
  • TIMER_ON：1 使能定时器，0 复位定时器。
  • ALARM：1 触发报警，0 清除报警。

2.2 分层设计中使用的内部信号

通行检测子有限状态机为主有限状态机生成以下内部信号：

  • PASS_ACTIVE：乘客位于传感器检测区域内，或正在通行。
  • PASS_DONE：一次有效通行已经完成。
  • AREA_OCCUPIED：至少一个传感器被遮挡，即检测区域被占用。

主有限状态机输入

  • PAYTERM, TIME_OUT, PASS_ACTIVE, PASS_DONE, AREA_OCCUPIED, CLK, RESET

主有限状态机输出

  • GATE_OPEN, TIMER_ON, ALARM

通行子有限状态机输入

  • IR1, IR2, IR3, CLK, RESET

通行子有限状态机输出

  • PASS_ACTIVE, PASS_DONE, AREA_OCCUPIED

3.(b) 状态图与状态说明

3.1 主有限状态机

状态定义

  • A：待机状态，未接受已付款乘客，闸门关闭，定时器复位，无报警。
  • B：等待进入状态，已接受支付，闸门打开，定时器运行，等待乘客进入。
  • C：通行状态，乘客已经进入并正在通过闸门，定时器继续运行。
  • D：报警状态，乘客仍在传感器检测区域内时 1 分钟已到，系统保持报警，直到检测区域变为空。

主有限状态机状态图（Moore 型）

主有限状态机在各状态下的输出

  • A: GATE_OPEN=0, TIMER_ON=0, ALARM=0
  • B: GATE_OPEN=1, TIMER_ON=1, ALARM=0
  • C: GATE_OPEN=1, TIMER_ON=1, ALARM=0
  • D: GATE_OPEN=1, TIMER_ON=0, ALARM=1
在 B 中，如果定时器超时且无人进入，系统只需关闭闸门。 在 D 中，闸门保持打开，便于乘客离开；只有当传感器检测区域变为空，即 AREA_OCCUPIED=0 后，报警才会清除。

3.2 通行检测子有限状态机

为满足要求 (4) 到 (6)，传感器检测顺序必须从 IR1 逐步移动到 IR3。

有用的传感器分类

由 IR1 IR2 IR3 定义四个组合逻辑条件：

  • C0 = ~IR1 . ~IR2 . ~IR3：无传感器被遮挡，模式为 000
  • C1 = IR1 . ~IR3：乘客位于入口侧，有效模式为 100 或 110
  • C2 = IR2：乘客占据中间区域，有效模式为 010、110、111、011
  • C3 = ~IR1 . IR3：乘客位于出口侧，有效模式为 001 或 011

这可以支持不同乘客的预期移动模式：

  • 儿童乘客：100 -> 010 -> 001 -> 000
  • 成年乘客：110 -> 111/010 -> 011 -> 000
  • 携带行李的乘客：100 -> 110 -> 111 -> 011 -> 001 -> 000

像 101 这样非相邻传感器同时被遮挡的无效模式，不会被视为成功通行。

通行子有限状态机状态

  • P0：检测区域为空。
  • P1：入口侧先被占用，说明乘客从正确方向开始通行。
  • P2：乘客位于通道内部。
  • P3：乘客正从出口侧离开。

通行子有限状态机状态图（Mealy 型）

通行子有限状态机输出

  • 在 P1、P2 和 P3 中，PASS_ACTIVE=1。
  • 当 C0=0 时，AREA_OCCUPIED=1；也就是说，只要任一红外传感器被遮挡，检测区域就被占用。
  • 只有在有效转移 P3 -> P0 上，PASS_DONE=1。

这保证了只有乘客从 IR1 侧进入，并最终从 IR3 侧离开时，系统才会识别为成功通行。

7.简要结论

本解答满足全部要求：

  • 无乘客时系统处于待机状态
  • 支付成功后闸门立即打开
  • 能够检测从 IR1 -> IR2 -> IR3 的有效正向移动
  • 支持儿童、成人和携带行李乘客的情况
  • 无人进入时超时则关闭闸门
  • 乘客仍在内部时超时则触发报警
  • 只有在传感器检测区域变为空，即 AREA_OCCUPIED=0 后才清除报警

English version follows.

1.Design approach

This solution uses a hierarchical FSM, matching the hint in the assignment. The complete system is split into three parts:

  • Main FSM: handles standby, gate control, timeout, and alarm.
  • Passage-detection sub-FSM: checks whether the passenger moves through the three IR sensors in the correct order.
  • 1-minute timer: treated as an external block controlled by TIMER_ON.

This is cleaner than forcing all behavior into one large homogeneous FSM.

2.(a) Inputs and outputs of the FSM

2.1 External system signals

Inputs

  • PAYTERM: 1 means payment successful, 0 means standby.
  • IR1, IR2, IR3: infrared sensor outputs.
  • TIME_OUT: output from the 1-minute timer.
  • CLK: clock input for synchronous FSM implementation.
  • RESET: reset input.

Outputs

  • GATE_OPEN: 1 opens the gate, 0 closes the gate.
  • TIMER_ON: 1 enables the timer, 0 resets the timer.
  • ALARM: 1 raises the alarm, 0 clears the alarm.

2.2 Internal signals used in the hierarchical design

The passage-detection sub-FSM generates these internal signals for the main FSM:

  • PASS_ACTIVE: passenger is in the sensing zone or currently passing.
  • PASS_DONE: one valid pass has been completed.
  • AREA_OCCUPIED: at least one sensor is shaded, meaning the sensing area is occupied.

Main FSM inputs

  • PAYTERM, TIME_OUT, PASS_ACTIVE, PASS_DONE, AREA_OCCUPIED, CLK, RESET

Main FSM outputs

  • GATE_OPEN, TIMER_ON, ALARM

Passage sub-FSM inputs

  • IR1, IR2, IR3, CLK, RESET

Passage sub-FSM outputs

  • PASS_ACTIVE, PASS_DONE, AREA_OCCUPIED

3.(b) State diagram and state description

3.1 Main FSM

State definitions

  • A IDLE: standby, no paid passenger accepted, gate closed, timer reset, no alarm.
  • B WAIT_ENTRY: payment accepted, gate open, timer running, waiting for a passenger to enter.
  • C PASSING: passenger has entered and is moving through the gate, timer still running.
  • D ALARM_STATE: 1 minute expired while passenger is still inside the sensing area, alarm asserted until the area becomes empty.

Main FSM state diagram (Moore type)

Main FSM outputs in each state

  • A: GATE_OPEN=0, TIMER_ON=0, ALARM=0
  • B: GATE_OPEN=1, TIMER_ON=1, ALARM=0
  • C: GATE_OPEN=1, TIMER_ON=1, ALARM=0
  • D: GATE_OPEN=1, TIMER_ON=0, ALARM=1
In B, if the timer expires and nobody has entered, the system simply closes the gate. In D, the gate remains open so the passenger can leave; the alarm clears only after the sensing area becomes empty, meaning AREA_OCCUPIED=0.

3.2 Passage-detection sub-FSM

To satisfy requirements (4) to (6), the sensing sequence must move from IR1 to IR3.

Useful sensor classes

Define four combinational conditions from IR1 IR2 IR3:

  • C0 = ~IR1 . ~IR2 . ~IR3: no sensor shaded, pattern 000
  • C1 = IR1 . ~IR3: passenger is at the entry side, valid patterns 100 or 110
  • C2 = IR2: passenger occupies the middle zone, valid patterns 010, 110, 111, 011
  • C3 = ~IR1 . IR3: passenger is at the exit side, valid patterns 001 or 011

This supports the expected movement for different passengers:

  • child: 100 -> 010 -> 001 -> 000
  • adult: 110 -> 111/010 -> 011 -> 000
  • passenger with luggage: 100 -> 110 -> 111 -> 011 -> 001 -> 000

Invalid non-neighbor patterns such as 101 are not treated as a successful pass.

Passage sub-FSM states

  • P0 CLEAR: sensing area empty.
  • P1 ENTRY_SIDE: entry side occupied first, so the passenger starts from the correct direction.
  • P2 MIDDLE_ZONE: the passenger is inside the passage.
  • P3 EXIT_SIDE: the passenger is leaving from the exit side.

Passage sub-FSM state diagram (Mealy type)

Passage sub-FSM outputs

  • PASS_ACTIVE=1 in P1, P2, and P3.
  • AREA_OCCUPIED=1 when C0=0, meaning at least one IR sensor is shaded.
  • PASS_DONE=1 only on the valid transition P3 -> P0.

This guarantees that a successful pass is only recognized when the passenger enters from the IR1 side and finally clears the IR3 side.

4.(c) ASM chart of the main FSM

A IDLE
  Outputs: GATE_OPEN=0, TIMER_ON=0, ALARM=0
  If PAYTERM=1 -> B
  Else -> A

B WAIT_ENTRY
  Outputs: GATE_OPEN=1, TIMER_ON=1, ALARM=0
  If PASS_ACTIVE=1 -> C
  Else if TIME_OUT=1 -> A
  Else -> B

C PASSING
  Outputs: GATE_OPEN=1, TIMER_ON=1, ALARM=0
  If PASS_DONE=1 -> A
  Else if TIME_OUT=0 -> C
  Else if AREA_OCCUPIED=1 -> D
  Else -> A

  Outputs: GATE_OPEN=1, TIMER_ON=0, ALARM=1
  If AREA_OCCUPIED=0 -> A
  Else -> D

This Gray-style assignment is convenient because the normal sequence is:

00 -> 01 -> 11 -> 10 -> 00

D X X X X 1 D 1 0 1

Then with D flip-flops:

D_Q1 = (~Q1.Q0.PA) + (Q1.Q0.~PD.(~T + O)) + (Q1.~Q0.O)

D_Q0 = (~Q1.~Q0.P) + (~Q1.Q0.(PA + ~T)) + (Q1.Q0.~PD.~T)

Main FSM outputs:

GATE_OPEN = Q1 + Q0
TIMER_ON  = Q0
ALARM     = Q1.~Q0

5.2 Passage sub-FSM state assignment

Use two flip-flops P1 P0:

State Meaning Code
P0 CLEAR 00
P1 ENTRY_SIDE 01
P2 MIDDLE_ZONE 11
P3 EXIT_SIDE 10

Passage sub-FSM transition table

Present state Condition Next state
P0 C1=1 P1
P0 otherwise P0
P1 C2=1 P2
P1 C0=1 P0
P1 otherwise P1
P2 C3=1 P3
P2 C0=1 P0
P2 otherwise P2
P3 C0=1 P0
P3 otherwise P3

With D flip-flops:

D_P1 = (~P1.P0.C2) + (P1.~C0)

D_P0 = (~P1.~P0.C1) + (~P1.P0.~C0) + (P1.P0.~C3.~C0)

Passage sub-FSM outputs:

PASS_ACTIVE = P1 + P0
PASS_DONE   = P1.~P0.C0
AREA_OCCUPIED  = ~C0 = IR1 + IR2 + IR3

These equations are the direct implementation of the chosen transition table for valid physical sensor patterns.

7.Concise conclusion

This solution satisfies all requirements:

  • standby when no passenger is present
  • gate opens immediately after successful payment
  • valid forward movement is detected from IR1 -> IR2 -> IR3
  • child, adult, and luggage cases are all supported
  • timeout with no entry closes the gate
  • timeout with passenger still inside raises the alarm
  • alarm is cleared only after the sensing area becomes empty, meaning AREA_OCCUPIED=0
